## 做完一批后的收尾清单

每完成一个岗位族（或修完一轮问题），按顺序打勾：

1. [ ] **算薪**：`python -m unittest tests.test_<族名> -v`，观察台 **算薪 → 汇总** 该族全绿
2. [ ] **Hub**：`python main.py compute --reconcile`，差异报告里该族 W–AI 行无红字
3. [ ] **登记**：`hub_performance.yaml` 补 `module` / `hub_columns`，写清备注
4. [ ] **字段拉通**（若需要）：`config/role_field_alignment/<族>.yaml` + `runtime.py` 分发
5. [ ] **测试**：补 `tests/test_*_performance.py`，必要时加 Hub W–AI 行级测试
6. [ ] **文档**：在 `docs/iterations/<族名>/README.md` 写收尾记录；**更新本文「当前进度」**
7. [ ] **提交前**：`python -m unittest discover -s tests -v`（或相关子集）

> 若中途为修 bug 打断：修完后从第 1 步重跑，不要假设上一轮结论仍成立。

### 跑批提速

| 你想… | 用… | 说明 |
|--------|-----|------|
| 改计算器 / 源 Excel / 绩效整理表逻辑 | `compute` 或 `compute --reconcile` | 必须全量重算 |
| **改单个岗位计算器** | `compute --from hub --only <role>` | 从 Hub 快照增量重跑 overlay（~2–5 分钟） |
| 只改 `parity_annotation.yaml`、批注规则、对账列配置 | `reconcile` | 跳过 compute，读已有 `提成汇总.xlsx` |
| 观察台看结果 | 先 `compute`，日常迭代用 `reconcile` | 完整跑批 ~10–17 分钟；仅对账 ~3–4 分钟 |

详见 [design/incremental-pipeline.md](./design/incremental-pipeline.md)。

`WorkbookLoader` 在单次 `compute` 内对同一 sheet 只读一次 Excel（如 `系统销售毛利`），减少销售顾问 overlay 重复 IO。
