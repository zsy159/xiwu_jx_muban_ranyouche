## v0.7.0 — 2026-07-09

### 新增

- **Hub 声明式规则引擎（生产路径）**：`HubMetricsRuleEngine`（F–P 全员指标）+ `HubRuleEngine`（销售顾问 / 销售主管 / 销售助理 W–AI）；配置入口 `hub_metrics_rules.yaml`、`hub_column_rules.yaml`、`hub_adjustment_rules.yaml`。`HubFormulaEngine` 拓扑逐格回放已从 `SalesPipeline.run()` 移除，仅保留对账标注与售后账套子类。见 [提成汇总-列级计算规则.md](iterations/hub/提成汇总-列级计算规则.md)。
- **多月可复用薪酬流水线（CLI 驱动）**：所有子命令支持 `--month YYYY-MM`（默认 `months_registry` 中 `2026-05`）；未知账期报错并提示 `onboard-month`。
- **`onboard-month`**：基于 `month.template.yaml` 生成 `month-<YYYY-MM>.yaml` 并写入 `months_registry.yaml`；`--inherit-topology <月>` 繁衍新月（继承 sales/rules/aftersales 拓扑 JSON，`golden_workbook=null`）；`--extract-topology` 从带公式销售账套反推拓扑至 `data/topology/<月>/`。
- **观察台「新月接入」页**：`pages/0_新月接入.py` — 上传底层表、注册账期、继承/反推拓扑，与 CLI `onboard-month` 对齐。
- **观察台「财务调账」页**：`pages/5_财务调账.py` — 双轨绩效整理表（`绩效整理表-系统生成.xlsx` / `绩效整理表-财务确认版.xlsx`）；下游 Hub overlay 与算薪优先读财务确认版。见 `pipelines/performance_sheet_paths.py`。
- **岗位扩展**：店端文员（`store_clerk`）、按揭专员（`mortgage_clerk`）、翼真新媒体等模块与配置；`canonical_column_registry.yaml`、`canonical_param_sheets.yaml` 统一列名与参数表。
- **财务合并账套模板**：`docs/templates/` 填表说明与 `scripts/generate_finance_*` 生成标准/可计算/合并工作簿。
- **Hub / 发薪列级文档**：`docs/iterations/hub/`（列级计算规则、底层表列使用清单、绩效整理表列映射）；`docs/iterations/payout/发薪表-岗位取数与计算逻辑.md`。
- **列来源追溯**：`commission_summary_column_sources.py`、`performance_sheet_column_sources.py`、`payout_column_sources.py` 支撑文档与测试。

### 变更

- 设计文档 [新月接入与Hub拓扑.md](design/新月接入与Hub拓扑.md) 改为 CLI 接入流程；[hub-绩效岗位分治.md](design/hub-绩效岗位分治.md) 同步销售顾问 W–AI 规则引擎状态。
- `months_registry.yaml` 预注册样板月 `2026-05`；移除独立 `month-2026-02.yaml` / `month-2026-04.yaml`（改由 `onboard-month` 按需生成）。
- `month.yaml` 精简为开发默认指针；`default_rules.yaml` 提供 canonical 拓扑与规则路径。
- 绩效整理表导出增加对账标色（`performance_sheet_formatting.py`）；`golden_perf_skips` 扩展跳过规则。
- 发薪上传 / 试算：`file_intake`、`sheet_merge`、`trial_run` 适配多月配置与默认规则；`summary_skeleton` 无金标准时行键回退策略。
- 销售顾问：`topology_specs` 完成率分组与 H 列封顶规则内化至 `hub_metrics_rules.yaml`；销售主管并入销售顾问 family，统一 W–AI 规则。

### 测试

- `tests/test_hub_rule_engine.py`、`tests/test_hub_metrics_rule_engine.py`、`tests/test_hub_adjustment_rule_engine.py`：规则引擎与金标准 parity。
- `tests/test_multi_month_cli.py`、`tests/test_onboard_ui.py`：`--month` / `onboard-month` / 新月接入 UI。
- `tests/test_performance_sheet_paths.py`、`tests/test_performance_sheet_reconcile.py`：双轨绩效表路径与对账。
- `tests/test_role_family_performance.py`、`tests/test_summary_skeleton_fallback.py`、`tests/test_topology_path_resolution.py` 等扩展。

## v0.6.4 — 2026-06-30

### 新增

- **非一线语义列**：管理岗（销售管理部/事业部/总经办）与支持部门（财务部等）在 `提成汇总` 增加岗位绩效、台次、提成系数等语义列；`non_frontline_roles.yaml` + `pipelines/non_frontline_columns.py`。

- 高亮列集扩展语义列 ↔ 物理列双向映射（`expand_highlight_columns`）。
- 观察台：`runOnSave=false`，试算长任务不被保存触发热重载打断。

### 测试

- `tests/test_non_frontline_columns.py`、`tests/test_commission_summary_formatting.py`；`test_parity_annotations` / `test_excel_format` / `test_parity_highlight` 扩展批注与语义列。

## v0.6 — 2026-06-30

### 新增

- **提成汇总对账四色高亮**：`reconcile` 输出的 `output/<月>/提成汇总.xlsx` 对差异与特殊格着色（浅灰 `#D9D9D9` 金标准直接填数；浅蓝 `#BDD7EE` 公式含手工；琥珀 `#FFEB9C` 数值不一致；浅橙 `#FCE4D6` 公式形态异常），**所有着色单元格均附带 Excel 批注**。
- **公式尾项常数自动检出**：`is_manual_formula_adjustment` / `collect_topology_manual_formula_cells` 识别 `=-140`、`=100`、`=SUMIFS(...)-100`、`=AH80-100` 等金标准手工调整，合并进 `build_reconcile_deferred_cells` 标浅蓝。
- **金标准直接填数检出**：`collect_topology_static_fill_cells` 从金标准 workbook 检出无公式静态格，标浅灰。
- 单元测试：`tests/test_parity_highlight.py`、`tests/test_parity_annotations.py` 覆盖高亮优先级、批注与登记合并。

### 变更

- `salary_pipeline/utils/excel_format.py`：统一 mismatch / deferred / static / anomaly 着色与批注写入；浅蓝优先于浅灰；deferred 优先于琥珀 mismatch。
- `salary_pipeline/calculators/sales_advisor/registry.py`：`build_reconcile_deferred_cells` 合并 `wa_parity_deferred`、二网 AH 留空、`topology` 公式尾项常数。
- `salary_pipeline/validation/parity.py`、`salary_pipeline/main.py`：对账流程串联高亮与批注步骤。
- `salary_pipeline/config/parity_annotation.yaml`：橙色个案与 deferred 登记调整；尾项常数个案改由浅蓝自动检出。
- 文档：`docs/iterations/sales-advisor/对账批注说明.md`、`WA五列对齐清单.md` 同步颜色图例与验收命令。

### 修复

- 同一格同时命中 static 与 deferred（含 `wa_parity_deferred`）时填充色与批注按约定以浅蓝为准。
- 已在 deferred 登记的格不再重复橙色 anomaly 批注。
