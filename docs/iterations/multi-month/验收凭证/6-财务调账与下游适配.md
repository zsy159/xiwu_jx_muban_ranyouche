## 双轨文件约定

| 文件 | 写入方 | 用途 |
|------|--------|------|
| `output/<月>/绩效整理表-系统生成.xlsx` | `compute` / `export-performance-sheet` | 系统重算底稿，**永不**被财务页覆盖 |
| `output/<月>/绩效整理表-财务确认版.xlsx` | 财务调账页保存 | 手工调整后下游 Hub overlay / 算薪读取 |

## 共享 helper

```python

def resolve_performance_sheet_path(month_config: dict) -> Path:
    confirmed = performance_sheet_output_dir(month_config) / CONFIRMED_PERF_FILENAME
    if confirmed.exists():
        return confirmed
    return resolve_system_performance_sheet_path(month_config)
```

## 手动验收步骤

1. `python main.py compute --month 2026-05` 生成 `绩效整理表-系统生成.xlsx`
2. 观察台 → **财务** → **财务调账**，选择账期，确认徽章为「系统生成底稿」
3. 在 `data_editor` 中新增一行或修改某顾问 `单台绩效`，点击「保存调整并提交至发薪流水线」
4. 文件系统检查：
   - `绩效整理表-财务确认版.xlsx` 含修改
   - `绩效整理表-系统生成.xlsx` 内容与 compute 后一致（未被覆盖）
5. `python main.py compute --month 2026-05 --from hub` 或发薪上传增量试跑 → overlay 使用确认版数值
