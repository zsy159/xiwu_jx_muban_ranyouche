## 设计思路

1. **真相来源是 yaml + 计算器，不是 topology。** `HubFormulaEngine` 已从生产路径移除，仅 `--reconcile` 对账只读。列是否「在用」以声明式规则为准。
2. **三层消费：**
   - **层 A — 绩效整理表**（订单/VIN 级）：明细表 → 整理表列 → 间接进 Hub
   - **层 B — Hub F–P**（全员，按姓名）：任务表 + 整理表汇总
   - **层 C — Hub W–AI / AK**（岗位 overlay）：规则引擎或岗位子表
3. **接入状态：** ✅ 生产已接入 · ⚠️ 仅对账/批注文案 · ❌ 登记了但生产未读
4. **维护约定：** 改规则 yaml 或 `calculators/performance_sheet/*.py` 时同步更新本文。

## 四、登记但未接入生产的 sheet ❌

以下 sheet 在 `sheet_registry.yaml` 或批注文案中出现，**当前 HubMetricsRuleEngine / HubRuleEngine / overlay 不读取**；对应 Hub 列应为空或手工填入。

| Sheet | 批注/旧文档声称用途 | 实际 |
|-------|---------------------|------|
| 渗透率 | L 列 | F–P 已改用整理表 K/AB；无 penetration_module |
| 指标汇总 | M–P 毛利等 | 无 yaml/生产代码引用 |
| 综合表 | Hub AM/AP | HubAdjustmentRuleEngine：综合项(L)、（已发放奖励）(J) SUMIF(B,姓名) |
| 重功超期+活动 | Hub AN（月度活动列名） | HubAdjustmentRuleEngine：SUMIF(Q,姓名)→X；列名见 `hub.activity_column_name` |
| 绩效整理表 AU | Hub AO 超期 | HubAdjustmentRuleEngine：SUMIF(P,姓名)→AU（computed_perf_frame） |
| 保客考核明细 | Hub AR | 同上 |
| 直营店交车 | Hub AN | 同上 |
| 翼真考核 | 翼真 AK | 同上 |
| 产品经理提成核算 | 拓扑回放 | 仅 reconcile |
| 销售管理岗提成依据 新标 | 拓扑回放 | 仅 reconcile |
