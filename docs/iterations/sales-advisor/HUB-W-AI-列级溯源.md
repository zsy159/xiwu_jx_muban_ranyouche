## 1. 读本文档前先知道的三件事

### 1.1 三层结构

```text
原始明细表（终端、保险、按揭…）
        ↓ 按每笔订单（VIN 码）计算
绩效整理表（约 500 行订单）
        ↓ 按「销售顾问」姓名汇总
提成汇总（51 名顾问，13 列绩效）
        ↓ 部分列再乘「完成率」
最终 Hub 绩效金额
```

### 1.2 字母列名只是 Excel 位置代号

下文格式统一为：**中文名（列字母）**。  
例如「整车绩效（W 列）」= 提成汇总表第 W 列，表头文字就是「整车绩效」。

### 1.3 同名不同表 — 最容易混淆

| 中文名相似 | 在哪张表 | 列字母 | 含义 |
|-----------|---------|--------|------|
| 整车绩效 | **提成汇总** | W | 顾问最终拿到的整车绩效（汇总后） |
| 单台绩效 | **绩效整理表** | AG | 每笔订单的整车绩效（汇总前） |
| 座位险提成 | **提成汇总** | AG | 顾问座位险绩效（汇总后） |
| 座位险绩效 | **绩效整理表** | AO | 每笔订单座位险（汇总前） |
| 整车超额 | **绩效整理表** | AH | 订单级权限结余，汇总后进 Hub「权限结余绩效」 |
| 权限结余绩效 | **提成汇总** | X | 顾问最终权限结余 |

## 3. 完成率三种版式

同一 Hub 列，不同顾问乘的完成率可能不同：

| 版式 | 怎么识别 | 整车绩效（W） | 加装/保险（Y、Z） |
|------|---------|--------------|-------------------|
| 个人完成率 | 整车绩效公式含「× 销量完成率（H）」 | 单台绩效合计 × **H** | 各自合计 × **H** |
| 门店块 | 整车绩效公式含「× 合并完成率（BA）」 | 单台绩效合计 × **BA** | 仍 × **H** |
| 保险追加常数 | 保险绩效公式含「+600」等 | 单台绩效合计 **不乘**完成率 | 保险合计 × H **+ 常数** |

样例顾问：**何宇**（个人完成率）；**唐鹏**（门店块）；**韩柏成**（保险 +600）。

## 4. 十三列完整溯源

下面每一列按固定结构说明：**Hub 怎么算 → 绩效整理表哪列 → 每笔订单怎么算 → 原始数据从哪来**。

---

### ① 整车绩效（W 列）

**Hub 公式（个人顾问）：**

```text
按姓名汇总「单台绩效（AG 列）」× 销量完成率（H 列）
```

**Hub 公式（门店块顾问）：**

```text
按姓名汇总「单台绩效（AG 列）」× 合并完成率（BA 列）
```

**订单级 — 单台绩效（AG 列）怎么算：**

```text
查「提成标准」得单价 × 台数（K 列）
特例：指标汇总部门 = 武侯自有店 且 车种 = 星越L → 固定 200 元/台
```

**原始数据来源：**

| 驱动项 | 来源表 | 来源字段 |
|--------|--------|---------|
| 车种 | 系统销售毛利 | 车种 |
| 销售渠道 | 系统销售毛利 | 销售渠道 |
| 部门 | 系统销售毛利 → 比对表 | 部门 → 指标汇总部门 |
| 台数 | 绩效整理表 | 台数（默认 1） |
| 提成单价 | 提成标准 | 2026年标准（按 部门×渠道×车型 匹配） |
| 销售顾问 | 系统销售毛利 | 销售顾问姓名 → 绩效整理表「销售顾问（P）」 |

**观察台：** 算薪 → 销售顾问 →「整车绩效明细」已支持订单级下钻 ✅

---

### ② 权限结余绩效（X 列）

**Hub 公式：**

```text
按姓名汇总「整车超额（AH 列）」—— 不乘完成率
```

**订单级 — 整车超额（AH 列）怎么算：**

```text
基于：渠道（D）、实际整车节约权限（AA）、
      以及同单的 单台绩效+加装绩效+保险提成+按揭提成（AG+AI+AJ+AK）

规则概要：
  · 渠道为分公司/网络部/二网/大客户 → 不算（空白）
  · 直营店店面/自有店 且 实际整车节约权限>0 → 整车超额 = 权限 × 20%
  · 否则若 (权限×14% + 上述绩效合计) < 150 → 补足到 150
  · 否则 直营店/自有店 → 整车超额 = 权限 × 14%
```

**原始数据来源：**

| 驱动项 | 来源表 | 说明 |
|--------|--------|------|
| 整车节约 | 系统超额 | 按 VIN 匹配 |
| 按揭权限扣除 | 按揭原表 | 按 VIN 汇总 |
| 实际整车节约权限 | 上式推导 | 整车实际节约 − 按揭扣除 |
| 渠道标签 | 指标汇总部门 + 销售渠道 | 公式推导 |

**观察台订单下钻：** 待做 ⏳

---

### ③ 加装绩效（Y 列）

**Hub 公式：**

```text
按姓名汇总「加装绩效（AI 列）」× 销量完成率（H 列）
```

**订单级 — 加装绩效（AI 列）怎么算：**

```text
若 台数 = 0  →  订单合计(含税) × 12%
若 台数 ≠ 0 且 (精品最低价 − 不提成精品) > 0
            →  (精品最低价 − 不提成精品) × 12%
否则 0
```

**原始数据来源：**

| 驱动项 | 来源表 | 来源字段 |
|--------|--------|---------|
| 订单合计(含税) | 系统销售毛利 | 订单合计(含税) |
| 精品最低价金额 | 系统销售毛利 | 精品最低价金额 |
| 不提成精品 | 装饰台账 | 多段按订单号汇总 |
| 台数 | 绩效整理表 | 台数 |

**观察台订单下钻：** 待做 ⏳

## 5. 总览图（中文）

```text
                         ┌─ 提成标准（2026年标准 / 政策追加）
系统销售毛利 ────────────┤
比对表 ──────────────────┤──→ 单台绩效 ──→ 整车绩效（W）
                         │
系统销售毛利 + 装饰台账 ──→ 加装绩效（AI）──→ 加装绩效（Y）
系统超额 + 按揭原表 ──────→ 整车超额（AH）──→ 权限结余绩效（X）
保险明细 ────────────────→ 保险提成 / 座位险 / 玻璃险
                         │      └──→ 保险绩效（Z）/ 座位险提成 / 玻碎险提成
按揭明细 + 按揭原表 ─────→ 按揭提成 / 盈利产品
爱车保 / 上户提成 / 置换服务 / 延保提成 / 重功超期+活动 / 二手置换+大客户
                         └──→ 各自 Hub 列（AA ~ AI 中对应项）
```
