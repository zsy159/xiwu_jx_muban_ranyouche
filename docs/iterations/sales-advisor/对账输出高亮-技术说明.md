# 提成汇总对账输出高亮 — 技术说明

**版本：** v0.6  
**日期：** 2026-06-30  
**用户文档：** [对账批注说明.md](./对账批注说明.md)

---

## 1. 目标

在 `python main.py reconcile`（或 `compute --reconcile`）生成 `output/<月>/提成汇总.xlsx` 时，将金标准与系统差异、以及「不宜直接判错」的特殊格在 Excel 中**可视化**，并保证**每个着色格都有批注**，便于业务复核与迭代修复。

---

## 2. 流水线入口

```text
main.py reconcile
  → salary_pipeline/validation/parity.py（列级/单元格级比对）
  → registry.build_reconcile_deferred_cells（暂缓格集合）
  → excel_format：
       add_commission_summary_color_legend（第 2 行图例）
       highlight_commission_summary_mismatches（琥珀）
       highlight_commission_summary_deferred_cells（灰 + 蓝）
       add_commission_summary_annotations（橙 + 个案 reason）
```

`deferred_cells` 在写入工作簿前合并三类来源（见 `registry.py`）：

1. `wa_parity_deferred`（YAML）
2. `erwang_blank_ah_deferred_cells`（金标准 D 列含「二网」致 AH 留空）
3. `collect_topology_manual_formula_cells`（金标准公式尾项常数 / 纯常数公式）

`static_cells` 来自 `collect_topology_static_fill_cells`（金标准无公式且有值的格）。

---

## 3. 颜色与优先级

| 优先级（后写覆盖填充） | 颜色 | 触发条件 | 批注模板要点 |
|------------------------|------|----------|--------------|
| 1 | 琥珀 `#FFEB9C` | 金标准数值 ≠ 系统数值 | `金标准与系统数值不一致` + 双方值与差 |
| 2 | 浅灰 `#D9D9D9` | 金标准无公式直接填数 | `金标准直接填数，无公式` |
| 2 | 浅蓝 `#BDD7EE` | deferred 或公式含手工 | `公式含手工录入，对账暂缓` |
| 3 | 浅橙 `#FCE4D6` | 公式形态异常（如 #REF!） | YAML/topology `reason` + 可选数值附注 |

**同格冲突：**

- deferred（含浅蓝）**优于** static（浅灰）与 mismatch（琥珀）。
- static **优于** mismatch（同格既静态又与系统不一致时标灰）。
- 已登记 deferred 的格**不再**叠加橙色 anomaly 批注（避免重复说明）。

---

## 4. 公式含手工判定（`topology_specs.py`）

`is_manual_formula_adjustment(formula: str) -> bool`：

- 纯常数公式：`=±数字`（如 `=-140`、`=100`）
- 纯算术无引用：`=189*20`、`=100*0.8` 等（无 A1 格引用、无 `绩效整理表!` 等 sheet 引用）
- 标准聚合公式后带尾项：`+600`、`-100` 等（如 `=SUMIFS(...)-100`）
- 单元格引用后尾项：`=AH80-100`

**不**视为手工：无尾项常数的 `=SUMIFS(...)`、含格引用的 `=W80*0.8` 等，仍参与正常公式对账。

含 `#REF!` 的公式不走浅蓝，由 `parity_annotations` + YAML 走橙色个案说明。

检出列范围与 hub 数据列一致（`HUB_COLUMN_MAP`：F–P、W–AI 等）。

## 5. 模块职责

| 模块 | 职责 |
|------|------|
| `utils/excel_format.py` | openpyxl 填充色、Comment 创建；mismatch / deferred / static / annotation 四类 API |
| `calculators/sales_advisor/topology_specs.py` | 从金标准 workbook 扫描 static fill、manual formula |
| `calculators/sales_advisor/parity_annotations.py` | 加载 `parity_annotation.yaml`，收集橙色格，排除 deferred |
| `calculators/sales_advisor/registry.py` | `build_reconcile_deferred_cells` 合并暂缓集合 |
| `config/parity_annotation.yaml` | 橙色个案 `hub_annotations`、`wa_parity_deferred` 登记 |

## 7. 扩展与维护

- 新业务暂缓：优先 `wa_parity_deferred`（浅蓝）；若仅为公式尾项常数，通常无需 YAML，topology 会自动检出。
- 新橙色个案：仅用于 #REF! 或需文字说明的业务语境（`parity_annotation.yaml`）。
- 修复后：从 deferred 删除登记；金标准公式恢复标准形态后，浅蓝自动检出也会消失。
