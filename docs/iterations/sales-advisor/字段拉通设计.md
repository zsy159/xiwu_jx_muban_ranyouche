# 销售顾问字段拉通设计

**账期样板：** 2026-05  
**状态：** Phase D 原始数据层首版（整车绩效订单级）+ 汇总层填写 → Hub W–AI 复算  
**最后更新：** 2026-06-26

> **列级溯源（中文版）：** 十三列 Hub 绩效从原始明细到汇总的完整链路，见 [HUB-W-AI-列级溯源.md](./HUB-W-AI-列级溯源.md)（表头均用中文，列字母作括号备注）。

## 1. 与新媒体 / 邀约专员的差异

| 维度 | 新媒体 / 邀约 | 销售顾问 |
|------|--------------|----------|
| 数据来源 | 子表单元格直接填写 | **绩效整理表**按订单行重算后，按 P=姓名 **汇总** |
| 填写哲学 | 填可计数/可测量的**原始驱动项**，系统向上复算 | **两层**：原始数据层（订单级）→ 汇总层（按人 SUM） |
| 版式 | 主播 / 短视频 / 西物·超市·崇州 | **个人×H** / **门店块×BA** / **保险+常数** |
| Hub 写入 | 单列或少数列 | **W–AI 共 13 列**（对账门槛 6 列 + 扩展 7 列） |
| 子表 | 规则 sheet 有独立版式 | `销售提成标准` 164 条 ≠ Hub 49 可比行 |

销售顾问的「最终取数」不是子表某一格，而是 **绩效整理表列按人求和**，再经 Hub 公式乘完成率或加常数。

### 1.1 两层数据模型（2026-06-26）

| 层级 | 含义 | 界面 | 示例（整车绩效） |
|------|------|------|------------------|
| **原始数据层** | 订单级可编辑驱动项 | `8_销售顾问` → 展开「整车绩效明细」 | 车架号、车型、渠道、台数 → 提成标准 × 台数 |
| **汇总层** | 绩效整理表列按人 SUM | 表单「整车绩效合计」等 | AG 合计 → × 完成率 → Hub W |

原始数据层对齐新媒体哲学：**用户改驱动项，系统沿计算链向上汇总**，而不是直接改最终数。

## 2. 字段清单（最终取数层）

### 2.1 完成率乘数

| 字段 id | 界面标签 | 来源 | 公式作用 | 影响 Hub |
|---------|----------|------|----------|----------|
| `sales_completion_rate` | 销量完成率（H） | 提成汇总 H 列 | SUMIFS 结果 × H | Y/Z 加装保险；个人顾问 W |
| `store_completion_rate` | 店别完成率（BA） | 提成汇总 BA 列 | SUMIFS 结果 × BA | W 整车（门店块顾问） |
| `insurance_add_const` | 保险追加常数 | topology 解析 | SUMIFS(AJ)+常数 | Z（如韩柏成 +600） |

## 3. 数据流

```text
订单明细（系统销售毛利、保险明细…）
        ↓ Phase B
绩效整理表（逐 VIN 行，列 AG/AI/AJ…）
        ↓ 按 P=姓名汇总
        ├─ 原始数据层 ←── 订单级驱动项（整车：车型/渠道/台数/提成标准）
        └─ 汇总层     ←── 界面填写 perf_*_sum + 完成率
AdvisorAlignedInput（perf_*_sum + 完成率）
        ↓ topology 公式复算
Hub 提成汇总 W–AI
```

### 3.1 整车绩效计算链（已实现原始数据层）

```text
系统销售毛利（车型 D、渠道 E、部门 BL）
        ↓ 比对表
绩效整理表 A（部门摘要）、H/I
        ↓ 提成标准 lookup（店别 C × 渠道 D × 车型 E → F 列）
单车 AG = 标准 × 台数 K（星越L+武侯自有店特例 200×K）
        ↓ SUM by 顾问 P
perf_ag_sum
        ↓ × 销量完成率 H（或门店 BA）
Hub W 整车绩效
```

```text

│ 销量完成率（H）              │
└─────────────────────────────┘

│ AG 整车 | AI 加装 | AJ 保险  │
└─────────────────────────────┘

```

### 2.4 为何曾只显示六项？

| 范围 | 列 | 说明 |
|------|-----|------|
| **对账门槛（六项）** | W、Y、Z、AA、AB、AC | `hub_performance.yaml` 中 `parity_gate` 验收子集；流水线硬门禁只盯这六项 |
| **扩展七列** | X、AD–AI | 计算链路（topology + 绩效整理表汇总）早已支持，首版观察台为降低填写负担未展示 |
| **2026-06-26** | 全部 13 列 | 观察台已补全输入与对账表；门槛六项仍单独标注 |

## 7. 使用方式

1. 启动观察台：`streamlit run salary_pipeline/app/…`（或项目既有入口）
2. 侧栏 **算薪 → 销售顾问**
3. 选顾问 → **从当月 Excel 预填**（读绩效整理表汇总 + Hub 完成率）
4. 展开 **整车绩效明细**：查看每笔订单的车型/渠道/台数/提成标准；可编辑驱动项后 **应用到整车绩效合计**
5. 修改其他合计或完成率 → 点 **计算** → 查看十三列 Hub 绩效与金标准差异（六项门槛单独标注）
6. 全字段对照（含不适用灰显）→ **算薪 → 字段拉通 → 销售顾问**
