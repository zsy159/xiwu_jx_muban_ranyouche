## 1. 背景与数据链路

### 1.1 业务含义

**权限结余绩效**（提成汇总 **X 列**）衡量顾问在整车销售中「节约下来的权限」对应的绩效奖励。Hub 层**不乘完成率**，直接按姓名汇总订单级 **AH（整车超额）**。

### 1.2 数据链路（Hub X ← AH ← AA / X / V / Z）

```text
系统超额 V ──┐
经理权限 X ──┼──→ 实际整车节约权限 AA (= V + X − Z) ──┐
按揭扣除 Z ──┘                                      │
                                                     ├──→ 整车超额 AH ──SUMIFS(P,姓名)──→ Hub 权限结余绩效 X
单台绩效 AG ──┐                                      │
加装绩效 AI ──┼──→ perf_sum (= AG+AI+AJ+AK) ────────┘
保险提成 AJ ──┤
按揭提成 AK ──┘
```

**AH 标准规则**（与 topology 主公式一致，见 `from_closure.py::_compute_ah`）：

| 条件 | AH |
|------|-----|
| 渠道为分公司 / 网络部 / 二网 / 大客户 | 空白 |
| G 为月末**服务补录**品类（置换服务、上户收入等） | 空白（金标准预期；代码已修复，见 §2） |
| 直营店店面 / 自有店 且 AA > 0 | AA × **20%** |
| 否则若 AA×14% + perf_sum < **150** | **150 − perf_sum**（保底） |
| 否则（直营店 / 自有店） | AA × **14%** |

**经理权限 X：** 2026-05 样本月共 **59 笔 VIN** 手工录入常数 1000（YAML allowlist），使 Y = V + X、AA 相应抬高；该链路已与金标准对齐。

### 1.3 对账口径说明

| 层级 | 含义 |
|------|------|
| **订单级 AH** | 绩效整理表逐 VIN 的 AH 与金标准逐格比对 |
| **Hub X** | `SUMIFS(绩效整理表!AH, P, 姓名)` 与金标准提成汇总 X 列比对 |
| **W–AI 绩效层** | 13 列绩效块跟踪对账（不阻断 F–P） |
| **parity_gate** | 算薪岗位族 7 列硬门禁（含权限结余绩效） |
