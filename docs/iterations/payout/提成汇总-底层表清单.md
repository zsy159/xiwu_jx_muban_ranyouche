## 流水线如何消费这些表

```text
month.yaml（路径 + 开关）
        ↓
SummarySkeletonModule ──读金标准 提成汇总 仅 B/C/D 行键（店别/职务/姓名）
        ↓
PerformanceSheetModule ──读 A 类明细 → computed_perf_frame（绩效整理表）
        ↓
HubFormulaEngine ──topology 回放 F–P、部分 W–AI；WorkbookLoader 读工作簿单元格
        ↓
Overlay 模块（D 类子表）── 新媒体 / 邀约 / 客户 / 直营店经理 / 招聘 / 销售顾问
        ↓
apply_non_frontline_columns（非一线语义列；golden bootstrap 已 no-op）
        ↓
output/YYYY-MM/提成汇总.xlsx
```

**当前开关（2026-05，`month.yaml`）：**

| 开关 | 值 | 含义 |
|------|-----|------|
| `performance_sheet.use_computed` | `true` | 绩效整理表由 builder 重算 |
| `performance_sheet.use_golden_perf_sheet` | `false` | Hub 不读金标准整理表数值 |
| `hub.bootstrap_from_golden` | `false` | Hub 不回退读金标准提成汇总单元格 |
| `bootstrap_non_frontline_physical_columns` | **no-op** | 非一线列不再从金标准拷贝（`87ead909` 后） |

`WorkbookLoader`（`data_loader.py`）从 `workbooks.sales` 主账套按 sheet 名读单元格/列；可选 `sheet_sources_file` 将单表拆到独立 xlsx。

## A. 纯输入表（每月必须提供）

每月替换为当月业务数据。典型均在主销售工作簿内：

`data/raw/YYYY-MM/燃油车-YYYY年MM月西物超市销售提成(终).xlsx`

（文件名随财务导出变化；`month.yaml` → `workbooks.sales` 指向实际路径。）

### A1. 订单与成本核心（绩效整理表 + Hub 毛利块）

| Sheet | 用途 | Hub 依赖列 | 必须 | 典型表头行 |
|-------|------|-----------|:----:|:----------:|
| **系统销售毛利** | 订单骨架 O/G/P/K；闭包 H/I/R/L/S；渗透率上游 | F–P（间接）、W–AI（经整理表） | ✅ | 2 |
| **整车成本** | 指导价与五项返利 AW–BB；闭包 AH 权限链 | W–AI（经整理表） | ✅ | 1 |
| **工厂购进** | 整车成本公式上游（闭包边） | 间接 | ✅ | 1 |
| **系统二手车降价** | 系统销售毛利 INDEX 上游 | 间接 | ✅ | 1 |
| **终端明细表** | BC 终端返利（`SUMIFS` by VIN） | 间接 | ✅ | 1 |
| **比对表** | 系统部门 → 指标汇总部门（闭包 A/D） | W（经 AG 查表） | ✅ | 1 |

### A2. 保险 / 按揭 / 装饰

| Sheet | 用途 | Hub 依赖列 | 必须 | 典型表头行 |
|-------|------|-----------|:----:|:----------:|
| **保险明细** | 保险返利/提成/座位险/玻璃险 | Z, AG, AI（经整理表 AJ/AO/AP/AB） | ✅ | 1 |
| **按揭明细** | 按揭提成、吉致追加 | AA, AD（经 AK/AL） | ✅ | 1 |
| **按揭原表** | 盈利产品、整车超权限 Z | AD, X（经 AL/AH） | ✅ | 1 |
| **装饰台账** | 装饰成本 BH；不提成精品 → AI | Y（经 AI） | ✅ | 1 |

### A3. 任务 / 指标 / 考核补录

| Sheet | 用途 | Hub 依赖列 | 必须 | 典型表头行 |
|-------|------|-----------|:----:|:----------:|
| **销售任务及完成率** | 考核量 F、实际销量 G、完成率 H；集客/加装任务 | **F, G, H, I, J, K** | ✅ | 1 |
| **指标汇总** | 整车/加装/保险/按揭毛利 M–P；算术加项 | **M, N, O, P**；AM 等 | ✅ | 1 |
| **渗透率** | 保险渗透率 L 等（工作簿公式，尚未系统重算） | **L** 等 | ✅ | 1 |
| **翼真考核** | 翼真店管理岗整车完成考核 | **AK**（SUMIF） | 翼真店 | 1 |
| **保客考核明细** | 保客考核 | **AR** | 有保客行 | 1 |
| **直营店交车** | 直营店交车奖励 | **AQ** | 有直营行 | 1 |
| **综合表** | 综合项 AM、考核小计 AP 等 | **AM, AP** | ✅ | 1 |

> **说明：** `销售任务及完成率`、`指标汇总`、`渗透率` 在工作簿内含公式格，但月度**数值**须随账期更新；上传时视为「带公式的输入表」，系统读 **data_only** 结果参与 Hub 回放。

### A4. 服务类提成台账（绩效整理表 Slice 1–3 / 闭包）

| Sheet | 用途 | Hub 依赖列 | 必须 |
|-------|------|-----------|:----:|
| **上户提成** | 上户绩效 AN | **AC**（经整理表） | ✅ |
| **爱车保** | 爱车宝 AM | **AB** | ✅ |
| **置换服务** | 置换服务 AS → 并入上户 | **AC** | ✅ |
| **二手置换** （尾空格） | 二手车 AR 之一 | **AH** | ✅ |
| **大客户** | 二手车 AR 之二 | **AH** | ✅ |
| **延保提成** | 延保 AE | **AE** | ✅ |
| **重功超期+活动** | 特殊车型 AQ | **AF** | ✅ |
| **系统超额** | 权限链 V → AH | **X** | ✅ |

## 跨月外推指南

### 每月必须变更

| 项 | 位置 | 说明 |
|----|------|------|
| `month` | `month.yaml` | 如 `2026-06` |
| `workbooks.sales` / `rules` / `aftersales` | `month.yaml` | 指向 `data/raw/YYYY-MM/…` |
| `topology.sales` | `month.yaml` | 从新 xlsx 重提 `*.topology.json` |
| `outputs.*` | `month.yaml` | `output/YYYY-MM/…` |
| `performance_sheet.billing_month` | `performance_sheet_columns.yaml` | 与账期一致 |
| **A 类全部 sheet** | raw 主账套 | 当月业务数据 |
| **补录行 / VIN 别名** | `performance_sheet_columns.yaml` | 核对 `supplement_rows`、`p_overrides_by_vin` |
| **骨架行** | 金标准或上月模板 | 新人/离职调整 店别/职务/姓名 行 |

### 可沿用（规则未变时）

- `commission_rules/*.json`（提成依据未改）
- `hub_performance.yaml`、`sheet_registry.yaml`、`non_frontline_roles.yaml`
- 岗位子表 **结构**（列布局）；**数值**仍按月填
- 闭包拓扑 **模式**（重提 topology 后公式模式应一致）

### 推荐新月上线步骤

```bash
# 1. 放置 raw 工作簿，更新 month.yaml 路径
# 2. 提取拓扑
python scripts/extract_topology.py data/raw/YYYY-MM/燃油车-….xlsx

# 3. 闭包核对
python scripts/closure_report.py --anchor 提成汇总

# 4. 跑流水线
python main.py compute --reconcile

# 5. 可选：单独导出绩效整理表
python main.py export-performance-sheet
```

## Top 10 — 新月优先准备的输入

按 Hub 影响面与流水线硬依赖排序（先备这 10 张即可启动大部分销售顾问 F–P + W–AI 链路）：

| 优先级 | Sheet | 理由 |
|:------:|-------|------|
| 1 | **系统销售毛利** | 订单骨架 + 闭包上下文；无此表无法建整理表行 |
| 2 | **销售任务及完成率** | Hub F/G/H 硬门禁；完成率乘数 |
| 3 | **整车成本** | AW–BB 返利链 + 权限相关 |
| 4 | **保险明细** | 保险绩效 Z、座位险/玻碎险 |
| 5 | **按揭明细** + **按揭原表** | 金融 AA、盈利产品 AD |
| 6 | **装饰台账** | 加装绩效 Y |
| 7 | **提成标准** | 单台绩效 W 单价查表 |
| 8 | **指标汇总** | 毛利 M–P 与部分综合项 |
| 9 | **比对表** | 部门映射 → 提成标准维度 |
| 10 | **上户提成** / **爱车保** / **延保提成** | 上户 AC、爱车宝 AB、延保 AE 高频项 |

次优先：置换服务、二手置换、大客户、系统超额、重功超期+活动、终端明细表、翼真/保客/直营店考核、五张岗位子表。
