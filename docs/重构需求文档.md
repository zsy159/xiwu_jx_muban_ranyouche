# 汽车经销商薪酬结算系统 (Python Pandas) 重构需求文档

## 一、项目背景与目标
我们需要将现有的高度复杂、依赖大量跨表引用的 Excel 提成结算系统，重构为独立的 Python 计算流水线。系统包含三个核心账套：
1. **参数规则库** (`提成依据.xlsx`)：各岗位提成阶梯、系数、底薪标准。
2. **销售业务账套** (`西物超市销售提成.xlsx`)：包含整车销售、按揭、保险、装饰、精品、二手车等前端业务。
3. **售后业务账套** (`吉利超市售后提成.xlsx`)：包含机修、钣喷、配件、PDI、外拓理赔等后端业务。

**核心技术栈**：`Python 3.10+`, `Pandas`, `NumPy`, `OpenPyXL`。
**架构要求**：必须采用模块化设计，坚决做到“数据、规则、逻辑”三者分离。使用 Pandas 的向量化计算（避免使用 for 循环遍历行），通过 `merge` 和 `groupby` 实现多表关联。

## 三、核心业务逻辑梳理 (Cursor 需实现的业务代码)

### 1. 销售端计算流 (`modules/sales_calc.py`)
需要实现对销售顾问最终提成（`提成汇总`）的聚合，输入数据源包含多个子表，需通过“销售顾问姓名”或“订单号”进行 `pd.merge()`：
* **整车业务**：读取 `整车成本` 和 `终端明细表`，计算 `单车毛利 = 开票价 - 整车成本 - 折扣 + 厂家返利`。并根据库龄计算超期车扣款/重攻车型奖励。
* **增值业务 (衍生)**：
    * **按揭**：读取 `按揭明细`，根据贷款金额和期数（如48期/60期吉致金融）匹配服务费提成和金融绩效。
    * **保险**：读取 `保险明细`，区分新保/续保，提取商业险返点和交强险返点计算毛利。
    * **代办与装饰**：读取 `上户`、`延保提成`、`装饰台账` 和 `爱车保` 模块计算衍生毛利。
* **考核与结余**：提取 KPI 得分（渗透率考核、满意度考核），计算权限结余绩效，超权限部分予以倒扣。

### 2. 售后端计算流 (`modules/aftersales_calc.py`)
车间与售后服务顾问的计算以“产值”和“工时”为核心：
* **机电/钣喷车间**：读取 `5月车间产值`，计算公式为 `(正常工时 * 系数) + (保险工时 * 系数) + (索赔工时 * 系数)`。需扣除辅料耗用。
* **业务前台 (SA)**：读取 `05月业务提成`，计算提成基数 `业务产值 = 正常工时 + 正常材料 - 折扣 + 续保赠送`。按完成率阶梯匹配提成系数（如 α < 85% 系数为0，α ≥ 100% 系数为 0.018）。
* **外拓与续保**：读取 `保险专员5月外拓` 和 `05月续保产值及提成`，计算外协修理的返点和新转续/流失客户召回绩效。

### 3. 全局工资与扣款聚合 (`modules/payroll_merge.py`)
整合销售与售后的业务提成后，与基本工资表（`西物基本`、`超市基本`）进行合并 (Left Join)：
* **加项**：基本工资、级别津贴、车展补贴、交通通讯补贴。
* **减项**：读取 `公积金`、`社保表`、`服装扣款明细`、`工作车救急`。
* **个税计算**：基于当月应纳税所得额，自动计算个税，输出最终的 `实发提成` 和 `实发工资`。
